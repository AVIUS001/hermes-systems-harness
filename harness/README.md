# Hermes Systems Harness

Agentic systems engineering platform: explicit graph, change propagation, Hermes execution layer, Obsidian export.

## Quick start

```bash
./harness/scripts/init_local.sh

# Seed Hermes framework nodes (plus local bootstrap_extension.py if present)
python3 harness/scripts/bootstrap_hermes.py

# Demo impact analysis on the sample graph
python3 harness/scripts/impact_analysis.py \
  --node req-thermal-op-max \
  --summary "Max op temp 45→50°C" \
  --audit

python3 harness/scripts/export_obsidian.py
```

With private sources configured in `harness/config/sources.yaml`:

```bash
python3 harness/scripts/ingest.py
./harness/scripts/sync.sh
```

## What this is

An **execution harness** for AI agents doing real engineering work:

- Requirements, regulations, tests, artifacts, and compliance evidence are **nodes**
- Dependencies are **explicit edges**
- A change propagates through the model automatically
- Every artifact traces back to the requirement that created it

Built on three pillars:

1. **Obsidian systems graph** (`graph/` — gitignored until you export locally)
2. **Knowledge Base Vault** (this repo — framework + sample graph)
3. **G-Stack** — QA, review, ship on your platform repos

Hermes sits above those pillars as the execution dispatcher: graph state → bounded agent work packets → audit trails → human decisions.

Daily/weekly briefing emails are generated by `harness/scripts/memory_email_brief.py`. Draft mode writes to `harness/outbox/`; sending requires SMTP env vars, `--send --yes`, and `HERMES_BRIEF_RECIPIENT` (default `you@example.com`).
